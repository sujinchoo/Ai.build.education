Week 10 — GitHub, 코드의 타임머신

주제

Git/GitHub 기반 협업과 버전 관리 흐름을 실습 중심으로 익힌다.

학습 목표

  • add/commit/push/pull 명령을 정확히 사용할 수 있다.
  • 브랜치-PR 기반 협업 흐름을 설명할 수 있다.
  • 충돌(conflict) 발생 시 기본 해결 절차를 수행할 수 있다.

학습 내용 (목표 연계)

  • 기본 Git 명령 흐름: add -> commit -> push -> pull의 의미와 순서를 정확히 익힌다.
  • 브랜치-PR 협업: 기능별 브랜치에서 작업 후 PR로 리뷰를 받고 main에 병합하는 실무 흐름을 학습한다.
  • 충돌 해결 기초: 같은 파일을 동시에 수정했을 때 충돌 표시를 읽고, 의도에 맞게 수동 병합하는 절차를 익힌다.
  • 초급자 포인트: 작은 단위로 자주 커밋하면 충돌 추적과 롤백이 쉬워진다.

비주얼 콘셉트

브랜치 생성 → 기능 커밋 → 원격 푸시 → PR 생성/리뷰 → 머지

그림

gitGraph commit id: "main"; branch feature; checkout feature; commit id: "feat: update"; checkout main; merge feature; commit id: "merge PR";

학습 예시 및 코드

  • Git은 로컬 히스토리 관리 도구, GitHub는 원격 협업 플랫폼이다.
  • 실무 기본 루프는 branch -> commit -> push -> PR -> review -> merge다.
  • 커밋 메시지는 변경 의도를 명확히 기록해야 추적이 쉽다.
git checkout -b feature/login-ui
git add .
git commit -m "feat: 로그인 UI 문구 수정"
git push -u origin feature/login-ui
  • 최신 협업에서는 보호 브랜치, CI 검사, 코드오너 리뷰를 기본 정책으로 둔다.

핵심개념 정리

  • 버전 관리: 변경 이력 추적
  • 브랜치 전략: main 보호 + 기능 분기
  • PR: 코드 리뷰와 품질 게이트

실습 미션

  1. 이번 주 학습 목표 3가지를 확인하고, 각 목표를 검증할 수 있는 실습 항목을 최소 1개씩 수행한다.
  2. 실습 과정(입력값, 코드/설정, 실행 결과)을 문서나 노트에 정리한다.
  3. 어려웠던 점 1가지와 다음 주에 개선할 점 1가지를 작성한다.

확장 실습

  • 의도적으로 충돌을 만들고 수동 해결
  • Conventional Commits 형식 적용

체크리스트

  • [ ] PR 기반 협업 흐름을 설명할 수 있다.
  • [ ] 충돌 해결 절차를 실행할 수 있다.
