# Week 09 — API 활용과 프롬프트 엔지니어링 확장

## 학습 목표
- LLM API 요청/응답 구조를 작성하고 해석할 수 있다.
- 시스템/사용자 프롬프트 역할을 구분할 수 있다.
- 원하는 출력 형식을 안정적으로 유도하는 프롬프트를 설계할 수 있다.

---

## 1. API 호출의 기본 구조

대부분의 LLM API는 JSON 기반 요청을 사용합니다.

```json
{
  "model": "gpt-4.1-mini",
  "messages": [
    {"role": "system", "content": "너는 교육용 튜터다."},
    {"role": "user", "content": "이 문장을 3줄로 요약해줘."}
  ],
  "temperature": 0.3
}
```

---

## 2. 프롬프트 역할 분리

- System: 답변 톤, 제약, 정책
- User: 실제 작업 요청
- Assistant: 이전 맥락 유지

역할이 섞이면 출력이 불안정해질 수 있습니다.

---

## 3. 좋은 프롬프트의 원칙

- 과업을 명확히 쓰기
- 출력 형식을 지정하기
- 금지사항/제약조건 명시하기
- 예시(Input/Output) 제공하기
