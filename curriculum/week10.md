# Week 10 — GitHub, 코드의 타임머신 확장

## 학습 목표
- Git 기본 명령(add/commit/push/pull)을 정확히 사용할 수 있다.
- 브랜치와 PR 중심 협업 흐름을 설명할 수 있다.
- 충돌 발생 시 기본 해결 절차를 수행할 수 있다.

---

## 1. 버전 관리가 필요한 이유

코드는 계속 바뀌고, 팀 협업에서는 변경 이력 추적이 필수입니다.
Git은 "누가, 언제, 무엇을" 바꿨는지 기록하는 도구입니다.

---

## 2. 핵심 명령 흐름

```bash
git add .
git commit -m "feat: add login form"
git push
```

- `add`: 스냅샷 후보 선택
- `commit`: 로컬 이력 저장
- `push`: 원격 저장소 업로드
- `pull`: 원격 변경사항 동기화

---

## 3. 브랜치 전략

- `main`: 안정 버전
- 기능 브랜치: `feature/...`
- 수정 브랜치: `fix/...`

기능 단위로 브랜치를 나누면 리뷰와 롤백이 쉬워집니다.

---

## 4. Pull Request(PR)

PR은 단순 병합 요청이 아니라 코드 토론 공간입니다.

좋은 PR에는 다음이 포함됩니다.
- 변경 배경
- 변경 내용 요약
- 테스트 방법
- 영향 범위

---

## 5. 충돌 해결

충돌은 같은 파일의 같은 부분을 서로 다르게 수정할 때 발생합니다.

절차:
1) 충돌 파일 확인  
2) 의도에 맞게 수동 편집  
3) 재커밋 후 푸시

---

## 정리
- Git은 협업 안정성을 높이는 필수 도구다.
- 브랜치와 PR은 팀 개발의 기본 프로세스다.
