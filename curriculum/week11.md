# Week 11 — AI 서비스 구현 (MVP) 확장

## 학습 목표
- MVP 범위를 정의하고 우선순위를 설정할 수 있다.
- AI API 연동 시 장애 대응 로직을 설계할 수 있다.
- 보안 원칙(API 키, 입력 검증, 로그 마스킹)을 적용할 수 있다.

---

## 1. MVP 사고방식

MVP는 "최소 기능으로 핵심 가치 검증"이 목적입니다.
완성도가 아니라 학습 속도가 중요합니다.

### 질문
- 사용자가 가장 먼저 원하는 기능은 무엇인가?
- 없어도 되는 기능은 무엇인가?

---

## 2. AI API 통합 구조

기본 흐름:
1) 사용자 입력 수집  
2) 백엔드 전처리  
3) AI API 호출  
4) 결과 후처리 및 반환

이때 실패 시나리오를 반드시 설계해야 합니다.

---

## 3. 안정성 설계

- 타임아웃 설정
- 재시도(Exponential Backoff)
- Rate Limit 대응
- 사용자 친화적 오류 메시지

---

## 4. 보안 기본 원칙

- API 키는 `.env` 등 환경변수로 관리
- 사용자 입력 검증(길이, 포맷, 금지 패턴)
- 로그에 민감정보(키, 주민번호 등) 저장 금지

---

## 5. 실습 예시

"질문 요약 챗봇" MVP를 만들고,
- 정상 응답
- 빈 입력
- API 장애
세 가지 상황을 테스트합니다.

---

## 정리
- MVP의 핵심은 빠른 검증이다.
- 안정성과 보안은 초기부터 포함해야 한다.
